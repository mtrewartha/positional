# Global Tech Stack
This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they
relate to how it should handle global tech stack.

## When to use this skill

- When choosing which technologies to use for implementing new features
- When adding dependencies to ensure they align with the established stack
- When implementing UI components (should use Compose UI with Material Design)
- When working with databases (should use SQLite with Room ORM)
- When storing user preferences (should use DataStore)
- When writing tests (should use Kotest with BehaviorSpec style)
- When setting up CI/CD pipelines (should use GitHub Actions)
- When integrating monitoring or crash reporting (should use Firebase)
- When making any technology stack decisions across build.gradle.kts or project configuration files
