# Kotlin Library Modules
This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they
relate to how it should handle kotlin library modules.

## When to use this skill

- When creating README.md files that describe library module purpose and usage
- When versioning libraries using Semantic Versioning (SemVer) for published libraries
- When using Kotlin's explicit API mode to ensure public APIs are well-defined
- When documenting public APIs with KDocs to explain purpose and usage
- When hiding implementation details using internal visibility modifiers
- When minimizing external dependencies to reduce dependency conflicts for consumers
- When providing test fixtures in testFixtures source sets for dependent modules
- When providing ProGuard/R8 rules for library code shrinking requirements
- When maintaining backward compatibility and using deprecation annotations for API changes
- When working on any library modules in the core directory
