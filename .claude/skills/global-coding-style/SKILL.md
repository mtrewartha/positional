# Global Coding Style
This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they
relate to how it should handle global coding style.

## When to use this skill

- When writing any code and ensuring it follows established naming conventions
- When formatting code with consistent indentation, line breaks, and automated formatting tools
- When choosing meaningful, descriptive names that reveal intent instead of abbreviations
- When keeping functions small and focused on a single task for readability and testability
- When removing dead code, commented-out blocks, and unused imports
- When applying DRY principle to avoid duplication by extracting common logic
- When deciding whether backward compatibility code is needed (assume it's not unless specified)
- When working on any .kt files to ensure consistent style across the codebase
