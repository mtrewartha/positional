# Global Conventions
This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they
relate to how it should handle global conventions.

## When to use this skill

- When organizing files and directories in a predictable, logical project structure
- When maintaining README files with setup instructions, architecture overview, and contribution guidelines
- When creating commit messages, feature branches, and pull/merge requests with clear descriptions
- When handling environment configuration with environment variables instead of hardcoded values
- When managing dependencies, keeping them up-to-date and minimal with clear documentation
- When establishing code review processes with clear expectations
- When defining what level of testing is required before merging code
- When using feature flags for incomplete features instead of long-lived branches
- When maintaining changelogs or release notes to track changes
