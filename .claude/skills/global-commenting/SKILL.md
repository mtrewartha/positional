# Global Commenting
This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they
relate to how it should handle global commenting.

## When to use this skill

- When writing code that explains itself through clear structure and naming
- When adding minimal, concise comments to explain large sections of complex code logic
- When avoiding comments about recent changes, temporary fixes, or time-sensitive information
- When ensuring comments are evergreen and remain relevant far into the future
- When deciding whether code needs a comment or if better naming/structure would suffice
- When reviewing code and determining if existing comments add value or are redundant
- When working with any .kt files and making commenting decisions
