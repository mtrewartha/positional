# Kotlin KDocs
This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they
relate to how it should handle kotlin kdocs.

## When to use this skill

- When documenting public classes, functions, properties, or interfaces exposed by a module
- When writing KDoc comments with proper /** */ formatting and tags
- When ensuring documentation is concise and clear about what the API does
- When avoiding implementation details in public API documentation
- When documenting parameters, return values, and exceptions using KDoc tags
- When working with any public declarations in .kt files that need documentation
- When reviewing whether existing KDocs provide value or contain unnecessary implementation details
