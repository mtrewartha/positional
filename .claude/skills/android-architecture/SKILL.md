# Android Architecture
This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they
relate to how it should handle android architecture.

## When to use this skill

- When creating or restructuring Gradle modules and organizing them into app, feature, or core directories
- When implementing or refactoring UI using Compose and MVI (Model-View-Intent) architecture
- When designing repository patterns to abstract data access from local and remote sources
- When setting up Hilt for dependency injection across the application
- When making decisions about module visibility and dependency configuration
- When working on build.gradle.kts files to configure module structure
- When designing how different layers of the application (UI, domain, data) should interact
- When creating new features that need to follow the established architectural patterns
