# Android Testing
This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they
relate to how it should handle android testing.

## When to use this skill

- When writing unit tests in test source sets using Kotest
- When creating test doubles (fakes in testFixtures, mocks/spies with MockK)
- When testing Compose UI components with Compose UI testing framework
- When testing Kotlin Flows using the Turbine library
- When structuring tests with given/when/then sections following Gherkin format
- When naming test functions (givenX_whenY_thenZ for instrumented, backtick format for unit tests)
- When deciding between Robolectric and instrumented tests
- When ensuring tests are fast, isolated, and test behavior rather than implementation
- When using Kotest matchers and assertions with nullability contracts
